static

1.修饰的方法可以直接调用；

前言：数组：占用空间连续。 寻址容易， 查询速度快。 但是， 增加和删除效率非常低。
链表： 占用空间不连续。 寻址困难， 查询速度慢。 但是， 增加和删除效率非常高。

Collection

方法名 说明
boolean add(E e) 添加元素
boolean remove(Object o) 移除指定元素
void clear() 清空所有元素
boolean contains(Object o) 判断是否存在指定元素
boolean isEmpty 判断集合是否为空
int size() 集合的长度，集合元素的个数

容器List

名字 作用
ArrayList 随机访问高效，插入式删除效率低
LinkedList 随机访问低，插入式删除效率高，另外加入了新的方法，支持栈，队列操作
HashSet 使用散列函数存储和获取元素，删除不按插入顺序，即元素的散列值
TreeSet 使用红黑树对元素排序
LinkedHashSet 采用链表维护元素的顺序，采用了散列和Hashset

Set方法是一个不包含重复元素的Collection不存在get，set等需要索引的方法

List的toArray方法

toArray():toArray()方法会返回List中所有元素构成的数组，并且数组类型是Object[]。还要注意一点就是，返回的数组是新生成的一个数组，也就是说，多次运行toArray()方法会获得不同的数组对象，但是这些数组对象中内容一样的。也就是说，toArray()返回的数组是安全的，你可以对它进行任意的修改，其原因就是List不会维持一个对该返回的数组的引.

public static void main(String[] args) {
    List<Integer> list = new ArrayList<>();
    list.add(1);
    list.add(2);
    Object[] objects1 = list.toArray();
    Object[] objects2 = list.toArray();
    System.out.println("objects1 == objects2 : "+(objects1 == objects2));
    objects1[1]=4;
    System.out.println("show objects1: "+ Arrays.toString(objects1));
    System.out.println("show objects2: "+ Arrays.toString(objects2));
    System.out.println("show list: "+list);
}

结果：

objects1 == objects2 : false
show objects1: [1, 4]
show objects2: [1, 2]
show list: [1, 2]

List遍历方法

get,迭代器，for….each

for(String str:s)
System.out.println(str);

全部代码：

package strive;

import java.util.*;

public class IteratorDemo {

	public static void main(String[] args) {
		Scanner scanner = new Scanner(System.in);
		List<String> list =new ArrayList<String>();
		System.out.println("输入名称（以quit结束）");
		while(true)
		{
			System.out.print("# ");
			String input=scanner.next();
			if(input.equals("quit"))
			{
				break;
			}
			list.add(input);
		}
		Iterator iterator =list.iterator();/////迭代器
		while(iterator.hasNext())
		{
			System.out.println(iterator.next());
		}
		for(String as:list)////for...each
		{
			System.out.println(as);
		}
		for(int i=0;i<list.size();i++)/////get方法
		{
			System.out.println(list.get(i));
		}
	}

}

容器Map

名字 作用
HashMap 通过散列保证访问高效,线程不安全， 效率高。 允许 key 或 value 为 null。
TreeMap 保存key处于排序状态
LinkedHashMap 保持元素插入顺序，通过散列保持访问高效

Map方法

方法 说明
Object put(Object key, Object value) 存放键值对
Object get(Object key) 通过键对象查找得到值对象
Object remove(Object key) 删除键对象对应的键值对
boolean containsKey(Object key) Map 容器中是否包含键对象对应的键值对
boolean containsValue(Object value) Map 容器中是否包含值对象对应的键值对
int size() 包含键值对的数量
boolean isEmpty() Map 是否为空
void putAll(Map t) 将 t 的所有键值对存放到本 map 对象
void clear() 清空本 map 对象所有键值对
